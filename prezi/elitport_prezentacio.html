Probléma & megoldás

Mit old meg a rendszer?

Üzleti probléma

  • Időpontkezelés manuális adminisztrációval lassú és hibára hajlamos.
  • Páciens oldalon hiányzik az önkiszolgáló, online foglalási élmény.
  • Nehéz összehangolni orvos, szolgáltatás és szabad idősáv kapcsolatát.

ElitPort megoldás

  • Online foglalás valós slotokra (nem szabad szöveges időpont).
  • Role-alapú hozzáférés és admin felület a fenntartható működéshez.
  • Foglalás utáni e-mail visszaigazolás automatikusan.

Backend

API, üzleti logika, perzisztencia

Technológia

  • Node.js + Express.js
  • Sequelize ORM
  • SQLite3 (dev/test)
  • MariaDB driver támogatás

Biztonság

  • JWT token ellenőrzés middleware-ben
  • bcrypt jelszó-hash
  • Role ellenőrzési middleware

Műveletek

  • User, Staff, Consultation CRUD
  • Slots kezelése és szűrése
  • Booking létrehozás / módosítás / törlés

API bizonyíték

Jelenlegi végpontstruktúra

Auth + Users

  • POST /api/register, POST /api/login
  • Users route-ok token-védettek
  • Jelszó módosítás külön endpointon

Staff

  • GET /api/staff és /api/staff/:id publikus
  • POST/PUT/DELETE token-védett

Consultations / Slots / Bookings

  • Consultation és Slot CRUD tokennel védve.
  • Bookings listázás tokenes, egyedi lekérdezés publikus.
  • Foglalás létrehozásához token szükséges.

A slide szövege közvetlenül az api.js route-definíciókból készült.
